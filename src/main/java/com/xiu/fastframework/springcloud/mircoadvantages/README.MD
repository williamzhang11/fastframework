# 微服务优缺点

## 1.优点

（1）每一个服务足够内聚，代码容易理解

（2）开发效率，一个服务只做一件事

（3）微服务能够被小团队单独开发

（4）微服务是松耦合的，是有功能意义的服务

（5）可以用不同的语言开发，面向接口编程

（6）易于与第三方集成

（7）微服务只是业务逻辑的代码，不会和 HTML、CSS 或者其他界面组合
	开发中，两种开发模式
	前后端分离
	全栈工程师
	
（8）可以灵活搭配,连接公共库/连接独立库

## 缺点

（1）分布式系统的负责性

（2） 多服务运维难度，随着服务的增加，运维的压力也在增大

（3）系统部署依赖

（4）服务间通信成本

（5)数据一致性	

(6)系统集成测试

(7)性能监控
